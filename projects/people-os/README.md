"# people-os\n\nThree excerpts from an AI-native people-operations toolkit, chosen to show the *supporting craft* — how the system is wired, orchestrated, and tested — without exposing the prompts or scoring rubrics that make it work.\n\n**Context:** see [../people-os.md](../people-os.md) for the project overview.\n\n## Stack\n\nPython 3.11+, [FastMCP](https://github.com/jlowin/fastmcp) (Model Context Protocol), [LangGraph](https://langchain-ai.github.io/langgraph/) for typed pipeline orchestration, Pydantic for validation, the Anthropic SDK, and `asyncio` throughout.\n\n## What each file shows\n\n- **`server.py`** — FastMCP composition. Nine tools and two resources registered through one standardized interface, grouped by concern with zero coupling between the server wiring and the capabilities it exposes. Stays under ~90 lines no matter how many tools the system grows; note the `scorecard://{job_id}` resource URI template.\n- **`screening_pipeline.py`** — a typed LangGraph `StateGraph` (parse → screen → decide). The state is a `TypedDict` whose `errors` field uses `Annotated[list[str], operator.add]`, so a degraded stage appends a non-fatal error and the run still completes instead of crashing. Includes async batch fan-out with deterministic, best-first result ordering.\n- **`synthetic_candidates.py`** — a seeded, stratified synthetic-data generator. It produces labeled candidate profiles across a controlled distribution (30% strong / 30% borderline / 25% weak / 15% non-traditional) so an eval suite can run reproducibly without ever touching real, PII-sensitive hiring data.\n\n## Deliberately omitted\n\nThis is a portfolio excerpt, not a replication blueprint. Left out on purpose:\n\n- **Prompt templates** — the resume-extraction and screening prompts are the product's moat and are not shared.\n- **Scoring rubrics** — the real weighted scorecard dimensions, signal examples, and the score→decision thresholds live in private modules; the pipeline's `screen_node` here is a transparent placeholder.\n- **The eval framework file** — its accuracy / precision / recall / consistency / fairness-gap design is described above, but the file itself embeds the rubric-driven output contract and a prompt-versioning scheme and stays private.\n- **Integrations & secrets** — ATS API clients, company context, credentials, and `.env` values.\n\nFiles here are faithful excerpts with light trimming and stubbed imports; identifiers and example data have been anonymized.\n\n_© 2026 Edoardo Caciolo — all rights reserved.
